What is an <iframe>?

An iframe (inline frame) embeds another HTML document inside your page. Think of it as a small window that shows content from another origin or resource.

Common uses: embedding videos, maps, forms, widgets, live previews, remote tools.
Security: use sandbox, set a descriptive title, and restrict permissions with allow when embedding third-party content.

Short History

Iframes became common in the late 1990s / early 2000s as a simple method to insert separate HTML contexts into a host page. Over time the web community added attributes and security patterns (like sandbox) to make embedding safer.

Late 1990s ‚Äî early embed techniques appear as websites start to include independent modules.
2000s ‚Äî widespread use for widgets and video players; cross-site risks become visible.
2010s+ ‚Äî sandboxing, CSP, and stricter browser policies improve iframe security.

How it works ‚Äî quick diagram

The parent page uses an <iframe src="...">. The iframe runs in a separate browsing context and loads its own document. Attributes control loading, permissions, and sandboxing.

<iframe src="https://example.com" title="Example" width="600" height="400"></iframe>

Pros & Cons

Pros
  • Easy embedding of third-party content
  • Isolates scripts & styles from the parent (somewhat)
  • Good for sandboxed tools and previews
Cons
  • Performance impact (extra document load)
  • Accessibility issues if title not provided
  • Security risks with untrusted content

What is an <iframe>?

An iframe (inline frame) embeds another HTML document inside your page â€” like a window to another site or resource.

Common uses: videos, maps, forms, widgets, previews.
Security: use sandbox, set title, and restrict permissions with allow.

Short History

Iframes became common in the late 1990s / early 2000s as a simple method to insert separate HTML contexts into a host page. Modern security options (sandbox, CSP) arrived later.

Late 1990s â€” early embed techniques appear.
2000s â€” widgets & video players use iframes widely.
2010s+ â€” sandboxing & CSP improve iframe safety.

How it works

The parent page uses <iframe src="...">. The iframe runs in a separate browsing context and loads its own document. Use attributes to control loading and permissions.

<iframe src="https://example.com" title="Example" width="600" height="400"></iframe>

Pros & Cons

Pros
  • Easy embedding
  • Separates context
  • Good for third-party widgets
Cons
  • Performance cost
  • Accessibility pitfalls
  • Security risks if untrusted
